## 一、数据模型概述

### 1.1 什么是数据模型？

在系统中，有各种「数据」，比如：
- 用户信息
- 演练会话记录
- 评估分数
- PPT 文件

这些数据不是散乱存放的，而是按照一定的「结构」组织起来的。这个「结构」就是数据模型。

### 1.2 核心实体关系图

下面这张图展示了系统中主要的数据实体以及它们之间的关系：

```mermaid
erDiagram
    USER ||--o{ PRACTICE_SESSION : has
    AGENT ||--o{ PRACTICE_SESSION : defines
    PERSONA ||--o{ PRACTICE_SESSION : plays
    PRACTICE_SESSION ||--o{ CONVERSATION_MESSAGE : contains
    PRACTICE_SESSION ||--o{ EVALUATION_REPORT : generates

    USER {
        string user_id PK
        string username
        string email
        string role
    }

    AGENT {
        string agent_id PK
        string name
        string category
        string status
    }

    PERSONA {
        string persona_id PK
        string name
        string category
        string difficulty
    }

    PRACTICE_SESSION {
        string session_id PK
        string user_id FK
        string agent_id FK
        string persona_id FK
        string status
        float logic_score
        float accuracy_score
    }

    CONVERSATION_MESSAGE {
        string message_id PK
        string session_id FK
        int turn_number
        string role
        string content
    }

    EVALUATION_REPORT {
        string report_id PK
        string session_id FK
        float total_score
        json details
    }
```

## 二、核心实体详解

### 2.1 用户（User）

用户是系统的基础，所有操作都围绕用户展开：

```mermaid
graph LR
    subgraph "用户属性"
        A[用户 ID] --> 用户
        B[用户名] --> 用户
        C[邮箱] --> 用户
        D[角色] --> 用户
        E[创建时间] --> 用户
    end
```

**用户角色**：
| 角色 | 权限 |
|------|------|
| 管理员 | 管理所有配置、查看所有数据 |
| 普通用户 | 使用演练功能、查看自己的报告 |

## 九、总结

核心数据实体：

| 实体 | 说明 | 关系 |
|------|------|------|
| **用户** | 使用系统的人 | 1 对 N 个会话 |
| **Agent** | 演练场景配置 | 1 对 N 个会话 |
| **Persona** | AI 角色设定 | 1 对 N 个会话 |
| **会话** | 一次演练 | N 对 1 用户/Agent/Persona |
| **消息** | 对话内容 | N 对 1 会话 |
| **报告** | 评估结果 | 1 对 1 会话 |

理解这些数据关系，可以帮助你更好地理解系统是如何运作的。
