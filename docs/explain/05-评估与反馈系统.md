## 一、评估系统概述

### 1.1 评估系统的价值

想象一下，你在健身房练健身，练完之后：
- **没有评估**：你不知道自己练得好不好，下次怎么改进
- **有评估**：教练告诉你力量提升了 5%，动作还有哪些问题

**评估系统就是那个「教练」**，它会告诉你：
- 这次演练表现如何？
- 比上次有进步吗？
- 哪些地方需要改进？

### 1.2 评估系统的两种模式

```mermaid
flowchart TB
    subgraph "评估模式"
        A[评估系统] --> B[实时评估]
        A --> C[综合评估]
    end

    subgraph "实时评估"
        B --> D[边练边评]
        D --> E[即时反馈]
    end

    subgraph "综合评估"
        C --> F[练后总结]
        F --> G[详细报告]
    end
```

| 评估模式 | 说明 | 例子 |
|---------|------|------|
| **实时评估** | 演练过程中即时反馈 | 刚说完话，马上告诉你说得怎么样 |
| **综合评估** | 演练结束后生成报告 | 全部练完后，给你一份详细报告 |

## 二、实时评估

### 2.1 什么是实时评估？

实时评估 = 在用户说话的同时，系统立即给出反馈

```mermaid
sequenceDiagram
    participant 用户 as 用户
    participant 系统 as 评估系统

    用户->>系统: 说了一句话

    rect rgb(240, 248, 255)
    Note over 系统: 立即分析
    系统->>系统: 1. 检测模糊词
    系统->>系统: 2. 识别销售阶段
    系统->>系统: 3. 计算即时得分
    end

    系统-->>用户: 即时反馈：
    Note over 系统,用户: "逻辑清晰 ✅"<br/>"建议：补充具体数据 💡"
```

### 2.2 实时评估的内容

```mermaid
mindmap
  root((实时评估内容))
    模糊话术检测
      检测到模糊表达
      建议优化方式
    销售阶段识别
      当前在哪个阶段
      阶段转换是否合理
    知识引用
      是否引用产品知识
      引用是否准确
    即时得分
      逻辑分
      准确分
```

### 2.3 实时反馈示例

```
用户说：我们这个产品效果很好...

系统立即反馈：
┌─────────────────────────┐
│ ✅ 逻辑清晰             │
│ ⚠️ "效果很好"有点模糊   │
│ 💡 建议：改成具体数据    │
│    例如：提升效率 30%    │
└─────────────────────────┘
```

## 三、综合评估

### 3.1 什么是综合评估？

综合评估 = 演练结束后生成一份完整的评估报告

### 3.2 报告生成流程

```mermaid

    subgraph "报告生成流程"
        A[用户结束演练] --> B[收集数据]

        C --> C2[知识运用]
        C --> C3[阶段覆盖]
    end

    style A fill:#e1f5fe
    style F fill:#e8f5e9
```

## 五、评分标准详解

### 5.1 逻辑性评分

```mermaid
flowchart TB
    subgraph "逻辑性评分标准"
        A[评分维度] --> B[表达清晰度]
        A --> C[逻辑连贯性]
        A --> D[结构合理性]

        B --> B1[优秀] --> 90-100
        B --> B2[良好] --> 70-89
        B --> B3[一般] --> 50-69
        B --> B4[较差] --> 0-49

        C --> C1[优秀]
        C --> C2[良好]
        C --> C3[一般]
        C --> C4[较差]

        D --> D1[优秀]
        D --> D2[良好]
        D --> D3[一般]
        D --> D4[较差]
    end
```

### 5.2 准确度评分

| 表现 | 得分 | 说明 |
|------|------|------|
| 完全准确 | 90-100 | 产品知识全部正确，无错误 |
| 基本准确 | 70-89 | 有少量错误，但不影响 |
| 有误 | 50-69 | 有明显错误，可能误导客户 |
| 错误多 | 0-49 | 错误较多，需要改进 |

## 七、分阶段评估

### 7.1 什么是分阶段评估？

分阶段评估 = 在特定时机触发评估，而不是等演练结束后

### 7.2 评估触发条件

```mermaid
mindmap
  root((评估触发条件))
    时间触发
      每 5 分钟评估一次
      演练时长达到 10 分钟
    轮次触发
      每 3 轮评估一次
      达到指定轮次
    关键词触发
      听到特定关键词
      提到竞品名称
    阶段触发
      销售阶段转换时
      从一个章节到另一个
```

### 7.3 评估触发流程

```mermaid
flowchart TB
    subgraph "触发评估流程"
        A[监控对话] --> B{满足触发条件？}
        B -->|否| A
        B -->|是| C[执行评估]
    end

    subgraph "触发条件"
        B --> T1[时间条件]
        B --> T2[轮次条件]
        B --> T3[关键词条件]
        B --> T4[阶段条件]
    end

    C --> D[生成阶段评估]
    D --> E[推送反馈]
```

## 八、反馈机制

### 8.1 反馈时机

```mermaid
stateDiagram-v2
    [*] --> 演练中: 用户说话

    演练中 --> 即时反馈: 说完一句话后
    即时反馈 --> 演练中: 继续对话

    演练中 --> 阶段反馈: 阶段转换时
    阶段反馈 --> 演练中: 继续对话

    演练中 --> 结束反馈: 用户结束
    结束反馈 --> [*]: 查看报告
```

### 8.2 反馈内容类型

```mermaid

      知识点补充

    阶段反馈
      阶段覆盖情况
      阶段表现评价
      阶段建议
    总结反馈
      整体得分
      主要优点
      主要问题
      改进方向
```

## 十三、总结

评估系统的核心价值：

1. **量化表现**：用分数直观展示演练效果
2. **即时反馈**：边练边改，效率更高
3. **历史追踪**：看到自己的进步曲线
4. **改进方向**：明确知道哪里需要提升

评估不是目的，而是帮助用户持续进步的工具。
