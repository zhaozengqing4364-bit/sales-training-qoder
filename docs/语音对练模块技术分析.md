# 语音对练模块技术分析

## 一、系统概述

本项目的语音对练模块是一个基于 **全双工 WebSocket** 的实时语音交互系统，主要用于销售话术训练场景。系统采用前后端分离架构，实现了从用户语音输入到 AI 语音回复的完整闭环。

### 核心设计原则

根据项目宪法 (Constitution)，系统遵循以下核心原则：

| 原则 | 目标 | 实现方式 |
|------|------|----------|
| **用户体验永不中断** | 零错误弹窗 | Result[T] 错误包装 + 优雅降级 |
| **实时性优先** | 端到端延迟 <300ms | 流式 ASR + 异步处理 |
| **容错与恢复** | 单点故障不崩溃 | 多级 Fallback 机制 |
| **成本控制** | 单次演练 <¥1 | Token 追踪 + 预算告警 |

## 三、核心模块详解

### 3.1 前端音频采集 (useAudioRecorder)

**文件位置**: `web/src/hooks/use-audio-recorder.ts`

#### 核心功能

1. **麦克风权限管理**: 请求并检测麦克风访问权限
2. **音频采集**: 使用 Web Audio API 的 `ScriptProcessorNode` 实时采集
3. **重采样**: 将浏览器原生采样率 (通常 48kHz) 转换为 16kHz
4. **格式转换**: Float32 → 16-bit PCM → Base64 编码

#### 关键代码

```typescript
// 重采样：线性插值算法
const resample = (inputData: Float32Array, inputSampleRate: number, outputSampleRate: number): Float32Array => {
    const ratio = inputSampleRate / outputSampleRate;
    const outputLength = Math.round(inputData.length / ratio);
    const output = new Float32Array(outputLength);
    
    for (let i = 0; i < outputLength; i++) {
        const srcIndex = i * ratio;
        const srcIndexFloor = Math.floor(srcIndex);
        const t = srcIndex - srcIndexFloor;
        output[i] = inputData[srcIndexFloor] * (1 - t) + inputData[srcIndexFloor + 1] * t;
    }
    return output;
};

// 音频处理回调 (每 4096 样本触发一次)
processor.onaudioprocess = (e) => {
    const inputData = e.inputBuffer.getChannelData(0);
    const resampledData = resample(inputData, 48000, 16000);
    const pcmData = floatTo16BitPCM(resampledData);
    const base64 = int16ArrayToBase64(pcmData);
    onAudioData(base64);  // 发送到 WebSocket
};
```

#### 技术要点

- **采样率**: 输出固定 16kHz (ASR 服务要求)
- **缓冲区大小**: 4096 样本 ≈ 85ms 延迟
- **音频格式**: 16-bit PCM, 单声道

### 3.3 后端 WebSocket 处理器 (EnhancedSalesHandler)

**文件位置**: `backend/src/sales_bot/websocket/enhanced_handler.py`

#### 核心职责

1. **连接生命周期管理**: 连接建立、心跳维护、断开清理
2. **流式 ASR 处理**: 实时接收音频并转发给 ASR 服务
3. **能力模块编排**: 并行执行模糊词检测、销售阶段识别、实时评分
4. **LLM 响应生成**: 基于 Persona 配置生成 AI 回复
5. **TTS 合成**: 将文本转换为语音并发送给前端

#### 消息处理流程

```python
async def handle_message(self, message: dict):
    msg_type = message.get("type")
    
    if msg_type == "audio_chunk":
        # 音频块 → 放入 ASR 队列
        await self._handle_audio_chunk(data)
        
    elif msg_type == "user_speaking":
        if data.get("speaking"):
            await self._start_streaming_asr()  # 开始流式 ASR
        else:
            await self._handle_audio_end()     # 结束并处理
            
    elif msg_type == "text":
        # 文本输入 → 直接处理
        await self._process_user_text(data.get("text"))
```

#### 流式 ASR 处理

```python
async def _run_streaming_asr(self):
    """运行流式 ASR 并处理结果"""
    asr_service = get_asr_service()
    
    async def audio_generator():
        """从队列生成音频块"""
        while True:
            chunk = await self.asr_queue.get()
            if chunk is None:  # 结束标记
                break
            yield chunk
    
    # 流式处理 ASR
    async for result in asr_service.stream_transcribe(audio_generator()):
        if result.is_success:
            self.current_transcript = result.value
            # 发送中间结果给前端
            await self._send_transcript(result.value, is_final=False)
    
    return self.current_transcript
```

#### 用户文本处理流程

```python
async def _process_user_text(self, text: str):
    """处理用户文本的完整流程"""
    
    # 1. 保存用户消息
    await self.message_storage.save_message(
        session_id=self.session_id,
        role="user",
        content=text
    )
    
    # 2. 并行执行能力模块
    capability_results = await self.capability_runner.run_all(self.context, text)
    
    # 3. 处理能力模块结果
    for result in capability_results:
        if result.data.get("detections"):
            await self._send_fuzzy_detection(result.data["detections"])
        if result.data.get("current_stage"):
            await self._send_stage_update(result.data)
        if result.data.get("overall"):
            await self._send_score_update(result.data)
    
    # 4. 生成 AI 回复 (带知识库上下文)
    response_text = await self._generate_response(text, knowledge_context)
    
    # 5. 保存 AI 消息
    await self.message_storage.save_message(
        session_id=self.session_id,
        role="assistant",
        content=response_text
    )
    
    # 6. TTS 合成并发送
    await self._send_tts_response(response_text)
```

### 3.5 LLM 服务 (OpenAI 兼容)

**文件位置**: `backend/src/common/ai/llm_service.py`

#### 核心特性

- **多 Provider 支持**: OpenAI, Azure, Anthropic
- **超时重试**: tenacity 库实现指数退避
- **成本追踪**: 按 session 统计 Token 消耗
- **Fallback 响应**: LLM 不可用时返回预定义话术

```python
@retry(stop=stop_after_attempt(2), wait=wait_exponential(min=2, max=10))
async def generate(self, prompt: str, session_id: str, system_message: str = None):
    try:
        messages = [SystemMessage(content=system_message)]
        messages.append(HumanMessage(content=prompt))
        
        result = await self._llm.agenerate([messages], callbacks=[cost_handler])
        
        # 成本追踪
        cost = (cost_handler.total_tokens / 1000) * self.cost_per_1k_tokens
        if cost > 0.8:  # 接近 ¥1 预算
            logger.warning(f"Session approaching budget: ¥{cost:.2f}")
        
        return Result.ok(result.generations[0][0].text)
        
    except Exception as e:
        # Fallback: 预定义响应
        return Result.fail(self._get_fallback_response(prompt, context))
```

async def synthesize(self, text: str) -> Result[AsyncIterator[bytes]]:
    try:
        communicate = edge_tts.Communicate(
            text,
            voice=self.voice,
            rate=self.rate,
            volume=self.volume,
            pitch=self.pitch
        )
        
        async def audio_stream():
            async for chunk in communicate.stream():
                if chunk["type"] == "audio":
                    yield chunk["data"]
        
        return Result.ok(audio_stream())

        # Fallback: 通知前端使用浏览器 TTS
        return Result.fail("[USE_BROWSER_TTS]")

### 3.7 能力模块系统 (CapabilityRunner)

**文件位置**: `backend/src/agent/capabilities/runner.py`

#### 设计模式

采用 **策略模式 + 组合模式**，支持动态配置和并行执行：

```python
class CapabilityRunner:
    def __init__(self, agent_config: dict, persona_config: dict = None):
        self.capabilities: list[BaseCapability] = []
        self._init_capabilities()
    
    async def run_all(self, context: AgentContext, input_data: Any) -> list[CapabilityResult]:
        """并行执行所有能力模块 (带超时保护)"""
        
        async def run_with_timeout(cap: BaseCapability) -> CapabilityResult:
            try:
                return await asyncio.wait_for(
                    cap.execute(context, input_data),
                    timeout=5.0  # 5秒超时
                )
            except asyncio.TimeoutError:
                return CapabilityResult(success=False, fallback="[CAPABILITY_TIMEOUT]")
        
        tasks = [run_with_timeout(cap) for cap in self.capabilities]
        return await asyncio.gather(*tasks)
```

#### 已实现的能力模块

| 模块 | 功能 | 触发条件 |
|------|------|----------|
| `fuzzy_detection` | 检测模糊用语 (大概、可能、应该) | 每轮对话 |
| `sales_stage` | 识别销售阶段 (开场→需求→方案→异议→成交) | 每轮对话 |
| `realtime_scoring` | 实时评分 (专业度、沟通技巧、产品知识) | 每轮对话 |
| `knowledge_retrieval` | 知识库检索 (RAG) | 每轮对话 |

## 五、错误处理与降级策略

### 5.1 多级 Fallback 机制

```
ASR 服务不可用
    └─▶ 返回 [USE_BROWSER_ASR] → 前端切换到 Web Speech API

LLM 服务超时
    └─▶ 返回预定义"垫场话术" → "请继续说明您的需求"

TTS 服务失败
    └─▶ 返回 [USE_BROWSER_TTS] → 前端使用 SpeechSynthesis API

WebSocket 断开
    └─▶ 指数退避重连 (最多 5 次) → 1s, 2s, 4s, 8s, 16s
```

### 5.2 Result[T] 错误包装

所有可能失败的操作都使用 `Result[T]` 包装，避免异常传播：

```python
class Result(Generic[T]):
    is_success: bool
    value: T | None
    fallback: str | None
    
    @staticmethod
    def ok(value: T) -> "Result[T]":
        return Result(is_success=True, value=value)
    
    @staticmethod
    def fail(fallback: str) -> "Result[T]":
        return Result(is_success=False, fallback=fallback)
```

## 八、总结

本语音对练模块通过以下技术实现了低延迟、高可用的实时语音交互：

1. **全双工 WebSocket**: 实现双向实时通信
2. **流式 ASR**: 阿里云 qwen3-asr-flash-realtime 提供 <200ms 延迟
3. **能力模块并行执行**: asyncio.gather 并行处理多个分析任务
4. **多级 Fallback**: 确保任何单点故障不影响用户体验
5. **Result[T] 错误包装**: 统一错误处理，避免异常传播
6. **ConfigManager**: 统一配置管理，支持热更新

系统设计充分考虑了实时性、可靠性和成本控制，适合企业级销售培训场景。
