 ### 📊 总结：按优先级排的行动建议

 P0 — 影响用户基本信任和使用：
 1. 🔴 首页硬编码用户名"亚历山大"→ 改读 currentUser
 2. 🔴 添加忘记密码/重置密码功能
 3. 🔴 侧边栏加入历史记录入口
 4. 🔴 新用户 Onboarding 引导（至少 3 步：选模式→选场景→开始练习）
 5. 🔴 训练中暂停/中断功能
 6. 🔴 Learner 侧 error boundary

 P1 — 影响使用体验和复练动力：
 7. 🟡 企业微信按钮要么接通要么移除/标注"即将支持"
 8. 🟡 报告导出（PDF/图片截图分享）
 9. 🟡 训练前场景预览说明页
 10. 🟡 Profile 页摆设功能（语速/通知）要么实现要么隐藏
 11. 🟡 首页所有空壳按钮（下载报告/设定目标/分享/筛选）要么实现要么移除
 12. 🟡 移动端底部导航（至少保证可访问核心功能）
 13. 🟡 统一意见反馈入口

 P2 — 增强粘性和运营效率：
 14. 🟢 训练路径/计划推荐
 15. 🟢 报告历史对比视图
 16. 🟢 Admin 批量导入用户
 17. 🟢 暗色模式
 18. 🟢 键盘导航和无障碍

 ────────────────────────────────────────────────────────────────────────────────

 核心判断：系统的训练核心链路（创建→练习→评分→报告→复盘→再练）已经扎实，后端证据链、评分语义、知识库检索可信度都做得比
 较深入。但前端 用户体验打磨明显落后于后端能力——大量 UI
 元素是空壳或硬编码，新用户缺乏引导，几个关键入口（历史记录、忘记密码）缺失。对于一个"小白用户"来说，第一次打开系统很可
 能在登录页或首页就卡住。
