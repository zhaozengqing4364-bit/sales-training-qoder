# CLAUDE.md Audit - 指令文件审计

递归扫描所有 CLAUDE.md 文件，检测冗余指令和优化机会。

## 执行步骤

### Step 1: 定位所有 CLAUDE.md 文件

扫描以下位置：
```
~/.claude/CLAUDE.md                    # 全局
<project>/.claude/CLAUDE.md            # 项目级（嵌套）
<project>/CLAUDE.md                    # 项目根
```

使用 Glob 查找：
```
**/CLAUDE.md
**/.claude/CLAUDE.md
```

### Step 2: 分析每个文件

对每个 CLAUDE.md 执行以下检查：

**2a. 基本统计**
- 总行数、总词数、章节数（## 标题数）

**2b. 冗余检测**
- 同一文件内重复出现的规则或指令（相似度 > 80%）
- 跨文件重复：全局 vs 项目级的重叠内容

**2c. 矛盾检测**
- 查找 "禁止" vs "必须" 等对立指令作用于同一主题
- 全局规则 vs 项目规则的冲突

**2d. 过长章节**
- 单个章节超过 50 行的，标记为 "可精简"
- 总文件超过 500 行的，建议拆分

**2e. 陈旧内容**
- 引用了不存在的文件路径
- 提到了过时的库版本或 API

### Step 3: 生成审计报告

```
## CLAUDE.md 审计报告

### 文件概览

| 文件路径 | 行数 | 词数 | 章节数 |
|----------|------|------|--------|
| ...      | ...  | ...  | ...    |

### 冗余检测

#### 文件内重复
- [文件A] 第 X 行与第 Y 行内容高度相似

#### 跨文件重复
- [文件A] 章节 "Code Style" 与 [文件B] 章节 "编码规范" 内容重叠 (N 行)

### 潜在问题

1. **过长章节**: [文件] 的 "项目结构" 章节有 XXX 行，建议提取为独立文档
2. **路径失效**: [文件] 引用的 `docs/xxx.md` 不存在
3. **版本过时**: [文件] 提到 Next.js 14 但项目使用 16

### 优化建议

1. 合并全局和项目级的重复规则
2. 将超长章节提取为独立文档并引用
3. 删除不再适用的历史记录
4. 统一术语（如 "禁止" vs "不允许" vs "绝对禁止"）
```

### Step 4: 输出可操作建议（新增）

对于每个问题，直接给出修复建议和具体操作：

```
## 可操作建议

### 问题 1: [具体问题描述]
- 问题: [问题详情]
- 建议: [修复方案]
- 操作: [直接可执行的命令或步骤]
  ```
  # 示例：
  删除 .claude/CLAUDE.md 第 X-Y 行的重复内容
  ```

### 问题 2: ...
```

### Step 5: 输出健康评分

- A (优秀): 无冗余，结构清晰，< 300 行
- B (良好): 少量冗余，结构合理，< 500 行
- C (需优化): 明显冗余或过长
- D (需重构): 大量冗余、矛盾或过长
