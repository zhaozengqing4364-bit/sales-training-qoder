# 实时语音识别 WebSocket 客户端事件参考

本文档介绍在与 Qwen-ASR Realtime API 的 WebSocket 会话中，客户端向服务端发送的事件。

> **用户指南**：模型介绍、功能特性和完整示例代码请参见 [实时语音识别-通义千问](https://help.aliyun.com/zh/model-studio/real-time-speech-recognition)。

## 1. session.update
用于更新会话配置，建议在 WebSocket 连接建立成功后，**立即发送此事件**作为交互的第一步。如果未发送，系统将使用默认配置。

服务端成功处理此事件后，会发送 `session.updated` 事件作为确认。

### 参数说明
| 参数                                            | 类型    | 是否必须 | 说明                                                         |
| :---------------------------------------------- | :------ | :------- | :----------------------------------------------------------- |
| `type`                                          | string  | 是       | 事件类型。固定为 `session.update`。                          |
| `event_id`                                      | string  | 是       | 事件ID。                                                     |
| `session`                                       | object  | 是       | 包含会话配置的对象。                                         |
| `session.input_audio_format`                    | string  | 否       | 音频格式。支持 `pcm` 和 `opus`。<br>默认值：`pcm`。          |
| `session.sample_rate`                           | integer | 否       | 音频采样率（Hz）。支持 `16000` 和 `8000`。<br>默认值：`16000`。<br>设置为 8000 时，服务端会先升采样到 16000Hz 再进行识别，可能引入微小延迟。建议仅在源音频为 8000Hz（如电话线路）时使用。 |
| `session.input_audio_transcription`             | object  | 否       | 语音识别相关配置。                                           |
| `session.input_audio_transcription.language`    | string  | 否       | 音频源语言。<br>支持：`zh`（中文）、`yue`（粤语）、`en`（英文）、`ja`（日语）、`de`（德语）、`ko`（韩语）、`ru`（俄语）、`fr`（法语）、`pt`（葡萄牙语）、`ar`（阿拉伯语）、`it`（意大利语）、`es`（西班牙语）等多种语言。 |
| `session.input_audio_transcription.corpus.text` | string  | 否       | 指定上下文（Context）。支持在语音识别的同时，提供背景文本、实体词表等参考信息，从而获得定制化的识别结果。<br>长度限制：不超过 10000 Token。 |
| `session.turn_detection`                        | object  | 否       | **VAD（语音活动检测）配置。**<br>它是启用/关闭 VAD 模式的开关：<br> - 若设为 `null`，则**关闭 VAD 模式**，启用 Manual 模式（需手动提交）。<br> - 若设为对象，则**启用 VAD 模式**。 |
| `session.turn_detection.type`                   | string  | 否       | 固定为 `server_vad`（当 `turn_detection` 存在时必须）。      |
| `session.turn_detection.threshold`              | float   | 否       | VAD 检测阈值。<br>默认值：`0.2`。<br>推荐值：`0.0`。<br>取值范围：`[-1, 1]`。<br>较低的阈值提高灵敏度（可能误判噪音为语音），较高的阈值降低灵敏度（有助于减少误触发）。 |
| `session.turn_detection.silence_duration_ms`    | integer | 否       | VAD 断句检测阈值（ms）。静音持续时长超过该阈值将被认为是语句结束。<br>默认值：`800`。<br>推荐值：`400`。<br>取值范围：`[200, 6000]`。<br>较低值（如300ms）响应更快但可能切断自然停顿；较高值（如1200ms）处理长句停顿更好但延迟增加。 |

### 示例
```json
{
  "event_id": "event_123",
  "type": "session.update",
  "session": {
    "input_audio_format": "pcm",
    "sample_rate": 16000,
    "input_audio_transcription": {
      "language": "zh",
      "corpus": {
        "text": "ASR语料，用以改进模型识别效果"
      }
    },
    "turn_detection": {
      "type": "server_vad",
      "threshold": 0.0,
      "silence_duration_ms": 400
    }
  }
}
```

## 2. input_audio_buffer.append
用于将音频数据块追加到服务端的输入缓冲区。这是流式发送音频的核心事件。

### 重要提示
*   **VAD 模式**：音频缓冲区用于语音活动检测，服务端会自动决定何时提交音频进行识别。
*   **非 VAD 模式**：客户端需配合 `commit` 事件使用。单个事件中的 audio 字段内容最大为 15 MiB。建议流式发送较小的音频块以获得更快的响应。
*   **无确认响应**：服务端不会对 `input_audio_buffer.append` 事件发送任何确认响应。

### 参数说明
| 参数       | 类型   | 是否必须 | 说明                                           |
| :--------- | :----- | :------- | :--------------------------------------------- |
| `type`     | string | 是       | 事件类型。固定为 `input_audio_buffer.append`。 |
| `event_id` | string | 是       | 事件ID。                                       |
| `audio`    | string | 是       | **Base64** 编码的音频数据。                    |

### 示例
```json
{
  "event_id": "event_2728",
  "type": "input_audio_buffer.append",
  "audio": "UklGRi4AAABXQVZFZm10IBAAAAABAAEAQB8AAEAfAAABAAgAZGF0YQAAAAA..." 
}
```

## 3. input_audio_buffer.commit
**仅在非 VAD 模式（Manual 模式）下使用。**
用于手动触发识别。此事件通知服务端，客户端已发送完一段完整的语音，将当前缓冲区内的所有音频数据作为一个整体进行识别。

*   **禁用场景**：VAD 模式（如果已配置 `turn_detection`，请勿发送此事件）。
*   **响应**：服务端成功处理后，会发送 `input_audio_buffer.committed` 事件作为确认响应。

### 参数说明
| 参数       | 类型   | 是否必须 | 说明                                           |
| :--------- | :----- | :------- | :--------------------------------------------- |
| `type`     | string | 是       | 事件类型。固定为 `input_audio_buffer.commit`。 |
| `event_id` | string | 是       | 事件ID。                                       |

### 示例
```json
{
  "event_id": "event_789",
  "type": "input_audio_buffer.commit"
}
```

## 1. error

### 参数说明

| `error.type`     | string | 错误类型。                                           |
| `error.code`     | string | 错误代码。                                           |

| `error.param`    | string | 与错误相关的参数。                                   |
| `error.event_id` | string | 与错误相关的事件ID。                                 |

### 示例
```json
{
  "event_id": "event_B2uoU7VOt1AAITsPRPH9n",
```

## 2. session.created
当客户端成功连接到服务端后，服务端响应的第一个事件。该事件包含服务端为此次连接设置的默认配置信息。

## 8. conversation.item.input_audio_transcription.text
此事件会高频发送，用于展示实时识别结果。

### 参数说明
| 参数            | 类型    | 说明                                                         |
| :-------------- | :------ | :----------------------------------------------------------- |
| `type`          | string  | 事件类型。固定为 `conversation.item.input_audio_transcription.text`。 |
| `event_id`      | string  | 事件ID。                                                     |
| `item_id`       | string  | 关联的对话项ID。                                             |
| `content_index` | integer | 包含音频的内容部分的索引。                                   |
| `language`      | string  | 被识别音频的语种（如 `zh`, `en`, `yue` 等）。                |
| `emotion`       | string  | 被识别音频的情感（如 `neutral`, `happy`, `sad` 等）。        |
| `text`          | string  | **已确认的文本前缀**。这是当前句子中，模型已确认不会再变更的部分。 |
| `stash`         | string  | **预识别的文本后缀**。这是紧跟在已确认部分之后，模型仍在处理、可能会被修正的临时草稿。 |

### 实时预览逻辑
在任何时刻，要获取当前最完整的句子预览，都需要将 `text` 和 `stash` 拼接起来：
**实时预览句子 = text + stash**

#### 示例：假设用户正在说：“今天天气不错，阳光明媚。”

| 时间点 | 用户说话进度 | API 响应 (text 和 stash)                        | 客户端UI 应显示 (text + stash)                    |
| :----- | :----------- | :---------------------------------------------- | :------------------------------------------------ |
| T1     | “今天……”     | text: "" <br> stash: "今天"                     | 今天                                              |
| T2     | “……天气……”   | text: "" <br> stash: "今天天气"                 | 今天天气                                          |
| T3     | “……不错”     | text: "今天" <br> stash: "天气不错"             | 今天天气不错 <br> *（“今天”已被确认并移入 text）* |
| T4     | （短暂停顿） | text: "今天天气不错，" <br> stash: ""           | 今天天气不错， <br> *（前半句完全确认）*          |
| T5     | “……阳光……”   | text: "今天天气不错，" <br> stash: "阳光"       | 今天天气不错，阳光                                |
| T6     | “……明媚。”   | text: "今天天气不错，" <br> stash: "阳光明媚。" | 今天天气不错，阳光明媚。                          |
| T7     | （结束说话） | -                                               | 使用 `completed` 事件的 `transcript` 作为最终结果 |

### 示例 JSON
```json
{
  "event_id": "event_R7Pfu8QVBfP5HmpcbEFSd",
  "type": "conversation.item.input_audio_transcription.text",
  "item_id": "item_MpJQPNQzqVRc9aC9zMwSj",
  "content_index": 0,
  "language": "zh",
  "emotion": "neutral",
  "text": "",
  "stash": "北京的"
}
```

# 实时语音识别交互流程

实时语音识别-通义千问服务通过 WebSocket 协议，接收实时音频流并实时转写。支持 **VAD 模式** 和 **Manual 模式** 两种交互流程。

## 1. 连接协议
建立 WebSocket 连接时，请使用以下 URL 和 Headers。

### URL
编码时，将 `<model_name>` 替换为实际的模型名称（例如 `qwen3-asr-flash-realtime`）。

```text
wss://dashscope.aliyuncs.com/api-ws/v1/realtime?model=<model_name>
```

### Headers
鉴权通过 HTTP Headers 进行。

```text
"Authorization": "bearer <your_dashscope_api_key>"
```

---

## 2. VAD 模式（默认）
**服务端语音活动检测（Voice Activity Detection）**。服务端自动检测语音的起点和终点（断句）。
开发者只需持续发送音频流，服务端会在检测到一句话结束时自动返回最终识别结果。
*   **适用场景**：实时对话、会议记录、语音输入法等。

### 启用方式
在 WebSocket 连接建立后的 `session.update` 事件中，**配置** `session.turn_detection` 参数（不为 null）。

![image](https://help-static-aliyun-doc.aliyuncs.com/assets/img/zh-CN/5617203671/CAEQYxiBgMDV65_y0xkiIGQ5YWQ3ZTIwN2RjZDQyZTlhMDI4YmFjYmIwMmIyY2Rj5812623_20251022102739.334.svg)

### 交互时序流程
1.  **语音开始**：服务端在检测到语音开始时，发送 `input_audio_buffer.speech_started` 事件。
2.  **音频发送**：客户端通过发送 `input_audio_buffer.append` 事件持续将音频追加到缓冲区。
3.  **语音结束**：服务端在检测到语音结束时，发送 `input_audio_buffer.speech_stopped` 事件。
4.  **自动提交**：服务端自动触发 `input_audio_buffer.committed` 事件，提交当前的输入音频缓冲区。
5.  **创建消息**：服务端发送 `conversation.item.created` 事件，包含从音频缓冲区创建的用户消息项。
6.  **返回结果**：服务端发送 `conversation.item.input_audio_transcription.text` 事件（实时中间结果）和 `completed` 事件（最终结果）。

---

## 3. Manual 模式
**手动断句模式**。由客户端控制断句逻辑。
客户端需要发送完一整句话的音频后，显式发送一个 `commit` 事件来通知服务端“这句话说完了”。
*   **适用场景**：客户端能明确判断语句边界的场景（如“按住说话”按钮、对讲机应用）。

### 启用方式
在 WebSocket 连接建立后的 `session.update` 事件中，将 `session.turn_detection` 参数**设为 `null`**。

### 交互时序流程

![image](https://help-static-aliyun-doc.aliyuncs.com/assets/img/zh-CN/5617203671/CAEQYxiBgID9.qzy0xkiIDhjMWZiMmQ2ZjQyMzQ0MzRiMjQyMmFjOTEwMmViZTU55812623_20251022102739.334.svg)

1.  **音频发送**：客户端通过发送 `input_audio_buffer.append` 事件持续将音频追加到缓冲区。
2.  **手动提交**：当客户端判断一句话结束时，发送 `input_audio_buffer.commit` 事件来提交缓冲区内的音频。
3.  **提交响应**：服务端发送 `input_audio_buffer.committed` 事件进行响应。
4.  **创建消息**：该提交操作会在对话中创建一个新的用户消息项（伴随 `conversation.item.created` 事件）。
5.  **返回结果**：服务端发送 `conversation.item.input_audio_transcription.text` 事件，包含语音识别结果。
