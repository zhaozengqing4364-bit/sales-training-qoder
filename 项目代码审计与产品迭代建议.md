一、项目整体评估

1. 代码结构概况
后端采用FastAPI + SQLAlchemy 2.0 + Pydantic 2.0，按场景拆分为sales_bot（销售对练）、presentation_coach（PPT演练）、agent（Agent平台）、admin（管理后台）四大业务域，共享common模块（音频、WebSocket、错误处理、知识库、AI配置等）。前端采用Next.js 16 + React 19 + TypeScript 5 + Tailwind CSS 4，App Router架构，分用户端（dashboard/practice）与管理端（admin）两套界面。

2. 核心业务流程概况
用户通过企微工作台进入，在dashboard选择演练场景（销售对练/PPT演练），进入practice页通过WebSocket与后端建立全双工语音交互。销售对练走StepFun Realtime双轨语音或legacy TTS降级链路；PPT演练走presentation_handler WebSocket。演练结束后生成comprehensive_report，用户在report页查看评分与证据。管理员在admin后台配置agent、persona、知识库、prompt、语音策略等。

3. 当前主要风险
第一，后端大量业务层service直接raise HTTPException，违反项目宪法"Result[T]取代异常"的强制规范，导致前端错误处理不一致。第二，StepFun实时链路通过5个超900行Mixin多继承拼装，总执行体超5000行，已超出可维护边界。第三，前端ErrorBoundary已实现但未在AppProviders或任何页面挂载，渲染级错误直接白屏。第四，admin端列表页CRUD模板零抽象，6个页面各自重复400-1000行样板代码。第五，ASR降级链只有熔断重试，无真实provider切换兜底。

4. 当前主要体验阻力
第一，WebSocket重连5次固定上限后放弃，弱网场景用户需手动重连，且无低频兜底重试。第二，知识库管理页搜索框/筛选Dialog/分页按钮均为占位UI，实际不可用。第三，演练报告页单文件2539行无锚点导航，用户查找评分维度需大量滚动。第四，dashboard首屏6个并发请求无AbortSignal、无防抖、无React Query缓存，触发多次重渲染。第五，401会话过期后1500ms才跳转，期间用户仍可点击导致401风暴。

5. 当前用户粘性机会点
第一，后端已有streakDays、weekly goal、growth、achievements等数据模型，但前端未做每日提醒/推送通道。第二，admin端已有"主管干预"完整闭环（INTERVENTION记录+提醒+结果回写），但未接企微/站内信推送。第三，report页已有highlight review本地持久化，但缺后端同步与"周回顾"召回。第四，dashboard已有effectiveness各维度pass_rate，但未驱动个性化推荐（当前为单条DEFAULT_RECOMMENDATION兜底）。第五，排行榜已存在但无成就解锁动画、无赶超提示、无分享卡片。

6. 基于当前提供的代码，暂无法确认的内容
第一，WebSocket心跳是否所有handler都强制30s超时（common/websocket/base_handler.py的完整实现未通读全部分支）。第二，知识库锁机制（KB Lock）的真实落点——components/stepfun_internal_knowledge_searcher.py仅为re-export，common/knowledge/kb_lock_guard.py未在本次范围内深读。第三，前端middleware.ts是否存在（web/src/middleware.ts未找到），路由守卫是否纯依赖server-auth.ts + useAuthProtection双轨。第四，后端是否对admin路由做独立API层鉴权（前端layout已通过requireServerSession拦截，但API层未在web端确认）。第五，recommendations/growth/analytics是否已在用户端dashboard真正展示（前端dashboard page调用getGrowth/getRecommendation，但推荐逻辑是否个性化未确认）。第六，AudioArchivalJob是否由外部cron/APScheduler触发（仓库内无调度定义）。第七，ConfigManager修改后admin写接口是否触发refresh_cache（admin写接口未审）。第八，prompts edit页是否已有试运行/diff能力（web/src/app/admin/prompts/[id]/edit/page.tsx未深读）。
