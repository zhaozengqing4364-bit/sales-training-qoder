# S03 UAT — 个人中心与排行榜体验收敛

## Preconditions
- 使用一个可正常登录的 learner 账号。
- 浏览器可访问当前 web 环境，并能进入 `/dashboard`、`/profile`、`/leaderboard`、`/forgot-password`。
- 至少准备一场可打开练习页并能听到 AI 播放音频的训练会话，用于验证语速偏好对真实播放链生效。
- 如需验证排行榜 fallback，请准备一个可让主榜单缺少 `my_rank` 字段的测试数据或 mock 环境。

## Case 1 — learner shell 的“编辑资料”入口必须进入真实 `/profile`
1. 登录 learner 账号后进入 dashboard。
2. 打开侧边栏底部的用户资料弹层（展开态）。
3. 点击 `编辑资料`。
4. 预期：浏览器进入 `/profile`，不是无反应按钮，也不是靠手动拼接 URL 才能访问。
5. 切换到 collapsed sidebar 模式，再次打开用户资料弹层并点击 `编辑资料`。
6. 预期：collapsed 模式下同样进入 `/profile`；即使 display name / email 缺失，也不会丢失 profile 路由入口。

## Case 2 — 个人中心的密码动作必须 truthfully hand off 到 forgot/reset 流程
1. 进入 `/profile`。
2. 在系统设置区找到“修改密码”。
3. 观察说明文案与 CTA。
4. 预期：说明文案明确写明“通过邮箱重置密码，沿用现有邮箱找回流程”；CTA 文案为“通过邮箱重置密码”。
5. 点击 CTA。
6. 预期：进入 `/forgot-password`，而不是停留在当前页、触发无反馈按钮、或使用浏览器硬跳转到其他未知地址。

## Case 3 — 语音播放速度偏好要在 `/profile` 中持久化
1. 进入 `/profile`，找到“语音播放速度”下拉框。
2. 依次确认可选项只包含 `0.75x`、`1.0x`、`1.25x`、`1.5x`。
3. 选择 `1.5x`。
4. 刷新页面或重新打开 `/profile`。
5. 预期：下拉框仍显示 `1.5x`；设置被保存在本地而不是刷新后丢失。

## Case 4 — 语速偏好必须真实影响练习页播放，而不是只改 profile 文案
1. 在 `/profile` 中把语速设置成 `0.75x`。
2. 进入一个可播放 AI 音频的练习页，触发一段 AI 语音播放。
3. 体感确认播放速度明显慢于默认值。
4. 返回 `/profile`，改成 `1.5x`。
5. 回到练习页再次触发 AI 语音播放。
6. 预期：新的播放明显更快；切换设置后无需重新保存 profile 资料，也不依赖后台设置接口。
7. 如环境支持不同播放路径（常规流式、fallback 音频、PCM），分别验证至少一次。
8. 预期：三种路径都遵守同一语速，而不是某一路保持默认 1.0x。

## Case 5 — 异常或脏本地存储值必须回退到默认 1.0x
1. 在浏览器开发者工具里把 `localStorage.voice_speed_preference` 手动改成非法值（如 `fast` 或 `2.0`）。
2. 刷新 `/profile`。
3. 预期：语速选择框自动回到 `1.0x`，页面不报错、不白屏，也不会保留非法值。

## Case 6 — 个人中心不应重新出现假的通知设置
1. 打开 `/profile` 的系统设置区。
2. 检查是否存在“通知开关”或类似无法影响真实行为的设置项。
3. 预期：系统设置区只保留真实会影响体验的语速偏好与密码重置入口；不会重新出现假的通知 toggle。

## Case 7 — 排行榜必须明确解释“只有可评估已完成训练计入排名”
1. 打开 `/leaderboard`。
2. 查看页面标题下方说明文案。
3. 滚动到底部查看页尾说明。
4. 预期：页头明确写出“均分与排名只纳入可评估的已完成训练，证据不足会话会单独记账，不会混入榜单”；页尾也明确说明证据不足会话不会拉高或拉低排行榜均分。
5. 确认页面上不再出现 weighted-score 一类旧语义表述。

## Case 8 — 排行榜筛选与 my-rank fallback 仍然工作
1. 在 `/leaderboard` 依次切换 `本周`、`本月`、`总榜`。
2. 再切换 `全部场景`、`销售对练`、`PPT 演练`。
3. 预期：筛选切换后榜单会重新加载，说明文案保持不变，仍然是 evaluable completed-session 语义。
4. 在一个主榜单缺少 `my_rank` 的测试环境/数据下重新打开页面。
5. 预期：页面会继续通过 fallback `getMyRank` 加载“我的排名”卡片；如果 fallback 也失败，则主榜单与说明文案仍正常显示，只隐藏我的排名卡片。

## Case 9 — 空榜单也要保留 truthful copy
1. 在空数据环境下打开 `/leaderboard`。
2. 预期：页面显示“暂无排行榜数据，完成可评估练习后会自动上榜。”
3. 同时确认页头与页尾关于可评估已完成训练的说明仍然保留，而不是在空态下退回旧语义或消失。

## Edge Checks
- **缺失 learner 字段：** 当当前用户缺少 display name 或 email 时，sidebar 用户弹层仍要保留进入 `/profile` 的入口。
- **profile 加载失败：** 即使 `/profile` 数据加载失败，页面也仍显示错误提示和“通过邮箱重置密码”链接，不能把密码操作一起打断。
- **跨刷新持久化：** 语速偏好修改后刷新页面、重新打开 profile、再进入练习页，值都应保持一致。
- **无后端新设置契约：** 操作语速下拉时，网络面不应出现新的 fake `voice_speed_preference` profile PATCH 依赖。
