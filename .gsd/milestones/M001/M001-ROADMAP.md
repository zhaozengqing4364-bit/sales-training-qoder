# M001: 桌面端销售训练闭环可用化

**Vision:** 把现有桌面端销售训练链路从“模块存在但整体偏演示、偏娱乐”收敛成一个可以真实给团队使用的训练闭环：客户演练多轮稳定、训练材料更新可生效、训练后报告可信、主管能据此做线下辅导、PPT 对练第一版具备完整会后复盘。

## Success Criteria

- 桌面端客户演练可以稳定完成多轮来回，不再频繁出现第二轮录音 / 第二轮响应 / 会话结束异常。
- 一次训练结束后，学员能看到可读、可信、可执行的单次报告，而不只是抽象分数。
- 主管查看单次报告时，能直接判断这次训练的结论、卡点、说虚 / 说错内容、未接住的异议以及下一次训练重点。
- 主管可以查看某个学员最近几次训练的连续变化，判断是否进步以及是否总卡在同类问题。
- 培训负责人或管理员更新公司标准 PPT / 产品资料后，下一次训练能够使用更新后的材料。
- PPT 对练第一版支持完整讲完后的统一复盘、评分与建议。

## Key Risks / Unknowns

- 多轮客户演练的运行时状态仍可能在第二轮或结束阶段失效 — 若主链路不稳，其它能力都没有上线意义。
- 报告、回放、趋势可能基于不同事实源 — 若事实不一致，学员与主管都不会信任结果。
- 知识库虽然可管理，但未必真实参与下一次训练 — 若材料更新不生效，训练内容会快速失真。
- 销售训练可能仍停留在泛泛对话，而无法围绕真实产品价值和异议处理展开 — 若训练目标不对，稳定也只是稳定地练错东西。
- PPT 对练可能已有页面或报告骨架，但没有围绕真实 PPT 价值点给出可执行复盘 — 若点评泛泛，会后复盘也不构成真实能力提升。

## Proof Strategy

- 多轮客户演练运行时稳定性 → retire in S01 by proving 桌面端真实客户演练可完成多轮往返、会话结束与重连场景可控。
- 报告 / 回放 / 趋势事实源不一致 → retire in S02 by proving 单次报告、回放与趋势读取引用同一会话事实基线。
- 材料更新不真实生效 → retire in S04 by proving 管理端更新公司标准 PPT / 产品资料后，下一次新建训练读取的是新材料并可诊断。
- 训练目标偏离真实价值表达与异议处理 → retire in S05 by proving 客户演练评价与提示围绕产品价值表达、客户收益与异议处理，而非泛泛闲聊。
- PPT 会后复盘泛泛无用 → retire in S07 by proving 完整 PPT 对练结束后输出的复盘能指出具体价值点、偏题 / 过长 / 错讲问题与下一次练习建议。

## Verification Classes

- Contract verification: 关键会话事实、报告 / 趋势 / 材料生效相关 API 与数据形状检查；关键模块测试与契约验证。
- Integration verification: 桌面端真实训练页 + WebSocket + 持久化 + 报告页 + 管理后台材料更新的端到端验证。
- Operational verification: 多轮会话、重连、结束、报告读取、材料更新生效与关键失败路径的日志 / 降级 / 诊断验证。
- UAT / human verification: 报告是否真的便于主管判断卡点与下次带教重点；PPT 复盘是否对真实业务表达有帮助。

## Milestone Definition of Done

This milestone is complete only when all are true:

- 所有 slice 都完成且关键用户路径能串成真实闭环。
- 多轮客户演练、单次报告、近期变化、材料更新生效、PPT 会后复盘都已真实接线，不是各自存在孤立页面或接口。
- 用户训练入口、管理材料更新入口、主管报告入口都存在且经过真实链路验证。
- 成功标准基于真实行为重查，而不是只基于文件存在或接口返回。
- 最终整合验收场景通过：桌面端真实客户演练与 PPT 对练都达到声明的证明级别。

## Slices

- [x] **S01: 多轮会话稳定化与运行时状态收口** `risk:high` `depends:[]`
  > After this: 桌面端客户演练可以稳定完成多轮来回，不再出现第二轮就坏、结束流程混乱、状态丢失等核心问题。

- [x] **S02: 训练证据落库与报告事实源统一** `risk:high` `depends:[S01]`
  > After this: 一次训练的转写、阶段、评分、模糊表达、关键异议等事实能被一致落库，报告与回放引用同一事实源。

- [x] **S03: 单次报告可读化（学员 + 主管）** `risk:medium` `depends:[S02]`
  > After this: 学员训练完能看懂结论和建议；主管打开单次报告能直接看到好坏结论、卡点、说虚 / 说错内容、没接住的异议、下次训练重点。

- [x] **S04: 知识库更新即生效链路** `risk:high` `depends:[S01]`
  > After this: 培训负责人 / 管理员更新公司标准 PPT 或产品资料后，下一次新建训练即可使用更新后的内容，并且能看到生效诊断。

- [x] **S05: 销售价值表达与异议处理基线** `risk:high` `depends:[S02,S04]`
  > After this: 客户演练不再停留在泛泛闲聊，而是围绕真实产品价值、客户收益、价格 / 竞品 / 证据类追问进行训练和判断。

- [x] **S06: 连续变化视图（主管判断是否进步）** `risk:medium` `depends:[S02,S03]`
  > After this: 主管能看到某人最近几次训练是否进步、反复卡在哪类问题、是否该切换训练重点。

- [x] **S07: PPT 对练会后统一复盘可用化** `risk:medium` `depends:[S02,S04]`
  > After this: 新人完成一轮 PPT 对练后，可以拿到围绕真实 PPT 价值点的统一复盘、评分和建议；先不要求实时打断。

- [ ] **S08: 桌面端发布验收与可观测性收口** `risk:medium` `depends:[S01,S03,S05,S06,S07]`
  > After this: 关键桌面端路径完成端到端验收，主要失败模式可定位、可回放、可诊断，达到“可以真实给团队用”的上线门槛。

## Boundary Map

### S01 → S02

Produces:
- 桌面端客户演练稳定的多轮会话生命周期约束（开始 / 多轮 / 结束 / 重连）。
- 统一的会话状态快照与恢复边界，供事实落库与报告生成依赖。
- 可复现的多轮稳定性验收路径。

Consumes:
- nothing (first slice)

### S02 → S03

Produces:
- 统一的训练事实记录形状：逐轮消息、阶段、评分、模糊表达、关键问题 / 异议证据。
- 报告、回放共享的会话事实读取入口与字段基线。
- “单次训练事实 → 报告视图” 的稳定映射边界。

Consumes:
- S01 的稳定会话生命周期与状态基线。

### S01 → S04

Produces:
- 稳定的新建训练 / 新会话切换边界，供材料更新后的下一次训练验证生效。
- 管理端与训练运行时之间不会因为状态脏数据导致材料版本错读的基础条件。

Consumes:
- nothing (first slice)

### S02 + S04 → S05

Produces:
- 面向销售价值表达与异议处理的训练判断基线。
- 基于真实材料的关键价值点 / 异议证据读取面。
- “训练目标围绕价值表达和异议处理” 的可验证评价输出。

Consumes from S02:
- 统一训练事实源。

Consumes from S04:
- 更新后真实生效的 PPT / 产品资料读取面。

### S02 + S03 → S06

Produces:
- 跨会话趋势聚合形状：近期变化、重复卡点、建议切换重点等。
- 从单次报告事实映射到近期变化视图的稳定边界。

Consumes from S02:
- 可比较的会话事实与指标基线。

Consumes from S03:
- 单次报告中的核心判断维度与问题分类。

### S02 + S04 → S07

Produces:
- 基于真实 PPT / 产品材料的 PPT 会后复盘事实读取与报告输出。
- 会后统一复盘的评价维度、偏题 / 过长 / 错讲问题说明与建议输出边界。

Consumes from S02:
- 统一训练事实源与报告基线。

Consumes from S04:
- 最新可用的 PPT / 产品资料事实源。

### S01 + S03 + S05 + S06 + S07 → S08

Produces:
- 桌面端首发验收路径：客户演练稳定、多轮报告可信、趋势可读、PPT 会后复盘可用。
- 关键失败模式的诊断与观测基线。
- “可真实给团队用”的最终发布证明。

Consumes from S01:
- 稳定会话生命周期。

Consumes from S03:
- 学员 / 主管单次报告视图。

Consumes from S05:
- 价值表达与异议处理判断基线。

Consumes from S06:
- 近期变化视图。

Consumes from S07:
- PPT 会后统一复盘能力。
