## 2. Improvement Law

Only promote work that does at least one of these well:

- makes the user journey clearer
- makes the core product loop easier to complete
- improves the quality of the main output or feedback
- increases future iteration safety through tests, contracts, diagnostics, or recovery

## 3. Current Evidence Snapshot

## 4. Priority Order

## 5. Candidate Scoring

- user leverage
- core-capability leverage
- evidence strength
- compounding value
- validation ease

Then subtract blast radius from 1-5.
