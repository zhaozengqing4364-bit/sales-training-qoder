# Land

## Preconditions

- The Linear issue is in `Merging`.
- `gh` CLI is authenticated.
- You are on the PR branch with a clean working tree.
- Human approval/merge intent is present.

## Workflow

1. Locate PR context:
   - `branch=$(git branch --show-current)`
   - `gh pr view --json number,url,title,body,mergeable,reviewDecision,statusCheckRollup`
2. Run a PR feedback sweep before merging:
   - `gh pr view --comments`
   - `gh pr view --json reviews`
   - `gh api repos/{owner}/{repo}/pulls/<pr>/comments`
3. Treat every actionable human or bot comment as blocking until fixed or explicitly answered with a justified pushback.
4. If the branch is stale or mergeability is conflicting/unknown, run the `pull` skill, fix conflicts, validate, commit, and push.
5. Run the strongest practical local gate for the PR scope. For broad changes prefer:
   - `bash scripts/critical-quality-gate.sh`
6. Watch GitHub checks:
   - `gh pr checks --watch`
   - If checks fail, inspect logs, fix, commit with `commit`, push with `push`, and repeat.
7. Squash merge only when review feedback is addressed and checks are green:
   - `gh pr merge --squash --subject "$pr_title" --body "$pr_body"`
8. Move the Linear issue to `Done` only after GitHub confirms merge.

## Review response rules

- Prefix GitHub comments generated by this agent with `[codex]`.
- For each review comment choose: accept, clarify, or push back.
- Reply with intended action before pushing code changes for that comment.
- Do not merge while unresolved actionable review comments remain.

## Output

Report merged PR URL, merge commit/squash result, checks observed, and Linear state update evidence.
